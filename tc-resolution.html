Resolution Patterns of Test Conflicts

For test conflicts, we manually check for the adopted resolution patterns. For failed test cases, our analysis is based on two kinds of changes:

  • Update on test case: we verify whether the test class and the failed test case are changed in the fix commit;
  • Exercised changes updated: the second verification looks for changes in the set of classes and methods changed that are involved in the test case failure. These changes can be in methods as also in the class structure. In case of changes in methods, only those involved in the failed test case execution are verified.
