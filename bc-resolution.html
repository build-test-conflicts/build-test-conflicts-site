Resolution Patterns of Build Conflicts

To check how a build conflict is fixed, our scripts use a syntatic diff for computing the changes between the merge commit of the broken build and its commit fix (merge-fix-diff, see here how syntatic diff is generated). Based in the information extracted from Travis log, we analize the involved classes looking for pre-defined resolution patterns.

Static Semantic Conflicts

Unavailable Symbol

Our scripts look for the folowing patterns:

  • Request symbol removal: in the Main class diff, our scripts look for the removal of the missing symbol: Delete SimpleName: {missing-symbol}
  • Request symbol Reintroduction: in the Secondary class diff, our scripts look for the addition of the missing symbol in the class using the message pattern: Insert SimpleName: {missing-symbol}
  • Reference update: update of the reference (missing symbol) for the new symbol identifier in the Main class diff using the pattern: Update SimpleName: {missing-symbol} to {new_symbol_name}.

Uniplemented Method

Our scripts look for the folowing pattern:

  • Method implementation: in the Main class diff, our scripts look for the pattern: Insert SimpleName: {missing-symbol} into MethodDeclaration

Incompatible Method Signature

Our scripts look for the folowing pattern:

  • Request removal: our scripts look in the Main class diff trying to match the following patterns into the diff: Delete SimpleName: {missing-method}

Duplicated Declaration

Our scripts look for the folowing pattern:

  • Duplication removal: our scripts try to match the following pattern in the diff of the Main class: Delete SimpleName: {duplicated_declaration}

Incompatible Types

Like the identification of conflicts of this category, the associated fixes are also done manually. Thus, we look for types changes in the request or in the operation that receives it. Additionally, we also consider the request removal of such operation.

Other Static Analysis Conflicts

Fixes can be checked looking the guidelines defined for the project. Thus, in our manual analysis we verify whether the set of files not following the project guidelines change their structure to adapt them for the styles conventions defined previously.
